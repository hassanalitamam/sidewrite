Token-saving features: these optimize costs in the Free-Tier Pool and during headless implement runs by reducing model output or compressing verbose tool output before the model sees it. Each feature is independently optional.
Terse replies
Free-Tier Pool providers answer more concisely, cutting output tokens on every implement/plan/review turn.
History compaction
Long conversations get deduplicated and truncated before dispatch (typically 85–95% smaller on long sessions) — the model can still retrieve the original content on demand if it needs it.
RTK command compression
Compresses noisy Bash tool output (git, grep, cargo test, …) before it reaches the model. Only has any effect if you separately install the third-party rtk CLI — sidewrite never installs it for you.
